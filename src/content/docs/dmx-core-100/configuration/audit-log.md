The audit log records all significant actions performed on the DMX Core 100, providing a history of who did what and when. This is useful for troubleshooting, security monitoring, and understanding system activity.

:::tip[Web UI only]
The audit log is available in the Web UI under **Utilities > Audit Log**.
:::

<!-- SCREENSHOT: Web UI audit log page -->

## Tracked Events

The audit log records events including:

- **Login/Logoff** — User sessions, including failed login attempts
- **Backup operations** — When backups are created or restored
- **Recording** — When recording is started and stopped
- **Configuration changes** — Modifications to settings, outputs, schedules, etc.
- **Playback events** — Cue, preset, and timeline playback actions

Each log entry includes:

- **Timestamp** — When the event occurred
- **User** — Who performed the action
- **Action** — What was done
- **Details** — Additional context about the event
